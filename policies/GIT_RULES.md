# GIT RULES

Règles Git globales :

1. Pas de commit automatique par défaut.
2. Pas de git push sans validation humaine.
3. Toujours lire le diff avant commit.
4. Toujours vérifier git status avant et après intervention.
5. Commits courts, traçables, explicites.
6. Une tâche = un diff maîtrisable.
7. Ne pas mélanger refactor, fonctionnalité et correction urgente.
