# PROJECT_RULES.md

## Règles spécifiques du projet

Compléter selon le projet.

## Fichiers sensibles

Ne jamais ajouter au contexte IA :

- .env
- fichiers credentials
- exports clients
- dumps base de données
- clés API
- tokens
- fichiers contenant secrets

## Règles de modification

- Une tâche à la fois.
- Diff limité.
- Pas de refactor massif sans story dédiée.
- Pas d'installation de dépendance sans validation.
- Pas de modification structurelle sans plan validé.

## Règles de test

Avant commit : compléter la commande de test projet.

## Règles Git

- Pas d'auto-commit.
- Pas de push.
- Commit local seulement après diff et tests.
